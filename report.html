The websites helped me to create designs for information and also what information should be displayed on my website.

Accessibility: State 3 ways in which your site is accessible to those with different abilities and needs (for example those with visual impairment)

- Those on mobile devices can view the website as the content is scaled for smaller devices. Users on Desktop or laptop devices can view the website also due to scaling media.

- Images and media have been compressed to minimise the amount of bandwidth needed, The Website needed an approx. of 256MB of data. After compression it only needs and approximation of 97MB - Thats more than 50%.

- The websites language is English, some browsers support multilingual translation such as the chrome browser, so I had to pick a language that would be able to be translated easily.

Usability: State 3 ways in which you considered the usability of your site.

- I wanted my website to be available for most devices to view, including mobile devices, as there are more mobile devices than local devices per person.

- Accessibilty is a top priority, especially for those with data and bandwidth caps. The website needed to be easy and fast to load , especially for high resolution images.

- Languages are important, as not everyone's native language is English, especially if you're a gamer or person living in Japan. A multilingual site was a necessity, so I had to decide on a language that could be translated by the browser.

Learning: state 3 things you had to learn or find out to create your site. How did you achieve that?

- This Module! (That was a joke yet true).

- I had to learn advanced techniques of HTML and their interactions with CSS and Javascript. I mainly needed to understand the use of span, form, document type interactions, and other ways to simplify my code.

- I also needed to learn different design techniques to use in HTML, such as blending and parallax backgrounds, gradients, image and local backgrounds, filters and other advanced design techniques.

- Compression and accessibility were some of the main things I needed to learn and improve. I had to compress images using compression software so they could load faster whilst also keeping their quality. I had to improve the layout for the mobile and desktop version without sacrificing quality and ease of access, done by designing UI that would suit both versions of the website in the most simplistic manner.

- Although I did not use Handlebars.js to its full capabilities, I did however implement in some places in my website. So I had to understand and learn the syntax (fairly simple), and understand how it works with the browser effectively.
